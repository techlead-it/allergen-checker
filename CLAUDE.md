# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## プロジェクト概要

高級旅館・ホテル向けの食物アレルギーチェックシステム（MVP）。仕入先情報の取込→アレルゲン正規化→レシピ紐付け→顧客アレルギーマッチング→判定リスト生成の5ステップワークフローをデジタル化する。

詳細な要件・設計は `docs/` ディレクトリ参照：
- `docs/DESIGN.md` - 要件定義・データ構造・判定ロジック
- `docs/FLOW.md` - 業務フロー図

## 画面構成と機能一覧

### ワークフロー画面（データ準備の3ステップ）

#### 1. 食材取込（`/import` - ImportPage）
仕入先から届いた規格書・ラベル・CSVを取り込み、食材情報を正規化してDBに反映する。

**機能**:
- ファイルアップロード（ドラッグ&ドロップ対応、PDF/画像/CSV/Excel）
- ファイル種別フィルタ（すべて/PDF/画像/CSV/Excel）
- アップロードキュー管理（OCR中→抽出済み→取込完了のステータス遷移）
- エラー時の再試行
- 抽出された食材の正規化確認（確定/要確認のステータス管理）
- アレルゲン自動検出・チップ表示
- 食材詳細モーダルでの原材料編集
- 全件確定後にDB反映

#### 2. レシピ管理（`/recipe` - RecipeLinkPage）
料理（レシピ）を作成し、使用する食材・調味料を紐づける。

**機能**:
- レシピ一覧（カードグリッド表示、食材数・プレビュー付き）
- 新規レシピ作成
- 食材リンク（2パネル構成: 利用可能な食材一覧 ↔ 紐づけ済み食材）
- 食材検索（名前フィルタ、紐づけ済みは除外）
- カテゴリ別グループ表示（主食材/調味料/共通仕込み）

#### 3. コース管理（`/course` - CoursePage）
コースを作成し、提供順にレシピを並べる。

**機能**:
- コース一覧（カードグリッド表示、料理数・プレビュー付き）
- 新規コース作成
- レシピ追加（2パネル構成: 利用可能なレシピ一覧 ↔ コース内レシピ）
- レシピ検索
- 提供順の並べ替え（↑↓ボタン）

### 管理画面

#### 4. ダッシュボード（`/dashboard` - DashboardPage）
顧客×コースの割当を一覧管理し、アレルゲン判定結果を俯瞰する中央管理画面。

**機能**:
- 割当一覧（デスクトップ: テーブル、モバイル: カード）
- 日付フィルタ・ステータスフィルタ（未確認/確認済/厨房共有済）
- 新規割当作成モーダル（顧客→コース→日付を選択）
- 割当削除（確認ダイアログ付き）
- 判定サマリ表示（OK/NG/要確認の件数）

#### 5. 割当詳細（`/dashboard/:assignmentId` - AssignmentDetailPage）
特定の顧客×コース割当に対して、料理ごとのアレルゲン判定確認とカスタマイズを行う。

**機能**:
- 料理ごとの判定結果表示（アコーディオン展開で食材詳細）
- 判定サマリチップ（NG/要確認/OK件数）
- ステータス更新（未確認→確認済→厨房共有済）
- 料理の差し替え（代替レシピ選択＋理由メモ）
- 料理の除外（コースから一時除外）
- 食材単位の除外（チェックボックスで個別除外→判定再計算）
- 食材名のインライン編集（オートコンプリート付き）
- コース変更（カスタマイズリセット警告あり）
- 厨房向けメモ入力

#### 6. 厨房シート（`/kitchen` - KitchenPage）
厨房スタッフ向けの読み取り専用画面。当日の調理指示とカスタマイズ内容を表示。

**機能**:
- 日付フィルタ（デフォルト: 当日）
- 印刷機能（印刷用CSS最適化）
- 「厨房共有済」割当のみ表示
- 折りたたみ可能なカード（顧客名/部屋/コース/アレルゲン表示）
- カスタマイズサマリバー（差し替え/食材変更/食材除外/料理除外の件数）
- 変更詳細（差し替え前後/変更前後の食材名/除外食材一覧）
- 厨房メモ表示

#### 7. 顧客管理（`/customers` - CustomerListPage / CustomerFormPage）
宿泊客のアレルギー情報を登録・管理する。

**機能**:
- 顧客一覧（カードグリッド、アレルゲンチップ・部屋名・チェックイン日表示）
- 新規登録・編集フォーム
  - 基本情報: 氏名（必須）、部屋名（検索可能セレクト）、チェックイン日
  - アレルゲン選択: 28品目＋カスタムアレルゲンのチェックボックス
  - 重症度: 微量NG/少量可/条件付き/不明
  - コンタミ許容: 不可/要確認/可
  - 自由記述・入力原文（監査証跡用）
- 削除（確認ダイアログ付き）

#### 8. アレルゲン管理（`/allergens` - CustomAllergenPage）
標準28品目の参照と施設固有のカスタムアレルゲンを管理する。

**機能**:
- 特定原材料8品目の一覧表示（赤バッジ、読み取り専用）
- 準特定原材料20品目の一覧表示（黄バッジ、読み取り専用）
- カスタムアレルゲンの追加・削除
- バリデーション（空入力/重複/28品目との重複を防止）

## ユースケース

### UC1: 食材情報の取込と正規化
**アクター**: 仕入/購買担当
1. 仕入先から届いた規格書・ラベル画像・CSVをアップロード
2. OCR/解析結果を確認し、食材名・原材料・アレルゲンを正規化
3. 要確認の食材を個別に確認して確定
4. 全件確定後にDB反映

### UC2: レシピの作成と食材紐づけ
**アクター**: 料理長
1. 新しいレシピ（料理）を作成
2. 利用可能な食材一覧から使用食材を検索・追加
3. 不要な食材を除外

### UC3: コースの作成とレシピ構成
**アクター**: 料理長
1. 新しいコースを作成
2. レシピを検索してコースに追加
3. 提供順に並べ替え

### UC4: 顧客アレルギー情報の登録
**アクター**: 予約/フロント担当
1. 宿泊客の氏名・部屋・チェックイン日を入力
2. アレルゲン（28品目＋カスタム）を選択
3. 重症度・コンタミ許容度を設定
4. 入力原文を記録（監査証跡）

### UC5: 顧客×コース割当の作成と判定確認
**アクター**: 料理長/サービス担当
1. ダッシュボードで新規割当を作成（顧客→コース→日付）
2. 割当詳細で料理ごとのアレルゲン判定結果（OK/NG/要確認）を確認
3. NG/要確認の料理に対してカスタマイズ（差し替え・食材除外・食材変更）
4. ステータスを「確認済」→「厨房共有済」に更新

### UC6: 厨房での調理準備
**アクター**: 厨房スタッフ
1. 厨房シートで当日の「厨房共有済」割当を確認
2. 各顧客のアレルゲン情報とカスタマイズ内容を確認
3. 印刷して調理場に掲示

### UC7: 施設固有アレルゲンの管理
**アクター**: 管理者
1. 28品目に含まれない施設固有のアレルゲンを追加
2. 不要になったカスタムアレルゲンを削除

## アーキテクチャ

### ルーティング (`src/App.tsx`)

HashRouterを使用（GitHub Pages対応）。3ステップのワークフロー画面と管理画面で構成：
- ワークフロー: `/import` → `/recipe` → `/course`
- 管理: `/dashboard`, `/kitchen`, `/allergens`, `/customers`

### 状態管理

バックエンドなし。各ドメインデータは `src/hooks/use*.ts` のカスタムフックで管理。パターンは共通：
- `useState` + `localStorage` による永続化
- 初期値は `src/data/mock.ts` のモックデータからロード
- hook例: `useRecipes`, `useCustomers`, `useCourses`, `useAssignments`, `useCustomAllergens`

### 型定義 (`src/data/types.ts`)

全ドメイン型をここに集約。主要な型：
- `ImportedIngredient` / `Ingredient` / `Recipe` - 食材・レシピ
- `Customer` - 顧客とアレルギー情報
- `Course` - コース（料理IDリスト）
- `CustomerCourseAssignment` / `DishCustomization` - 顧客へのコース割当とカスタマイズ

### 判定ロジック（ビジネスルール）

コアロジックは `src/utils/allergenCheck.ts`:
- `checkIngredient()`: 食材単位のアレルゲン判定
- `checkDish()`: 料理（レシピ）単位の判定（除外食材対応）
- 判定: **OK**（アレルゲン不含）/ **要確認**（`allergenUnknown: true`）/ **NG**（明確に含有）
- 特定原材料28品目リストは `src/data/mock.ts` の `allergen28Items`

`src/utils/resolveCustomizedDishes.ts` でコース内料理のカスタマイズ（差し替え・除外・変更）を解決。

### ツアーシステム (`src/tour/`)

初回利用者向けガイドツアー。`TourProvider` でコンテキスト管理、各画面のステップ定義は `src/tour/steps/` 配下。
